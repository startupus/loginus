# Быстрое исправление проблем

## Проверка

1. **ErrorPage** - исправлен импорт (default export)
2. **i18n режим** - установлен на 'dynamic' по умолчанию
3. **Статический импорт ErrorPage** - должен работать

## Возможные проблемы

Если страница не работает, проверьте в консоли браузера:

1. **Ошибки загрузки модулей** - проверьте Network tab
2. **Ошибки i18n** - проверьте, загружаются ли переводы через API v2
3. **Ошибки роутинга** - проверьте, правильно ли работает ErrorPage

## Что проверить

1. Откройте http://localhost:3000/ru/data
2. Откройте DevTools → Console
3. Посмотрите ошибки
4. Откройте DevTools → Network
5. Проверьте запросы к `/api/v2/translations/...`

## Если API v2 не работает

Временно можно переключиться на статический режим:
- В `frontend/src/services/i18n/v2/config.ts` измените дефолт на `'static'`
- Или создайте `.env.local` с `VITE_I18N_MODE=static`
