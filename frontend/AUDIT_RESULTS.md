# Результаты аудита дизайн-системы

## ✅ Что работает хорошо

1. **Глобальная тема** - ✅ Реализована через ThemeContext и CSS переменные
2. **API интеграция** - ✅ Все запросы идут через единый apiClient
3. **Мультиязычность** - ✅ i18n настроен, файлы переводов есть для всех модулей
4. **Темная/светлая тема** - ✅ Работает через CSS переменные и класс `dark`
5. **Структура Atomic Design** - ✅ Есть primitives, composites, layouts

## ❌ Проблемы, требующие исправления

### 1. Использование темы в компонентах

**Проблема**: Компоненты используют hardcoded классы типа `dark:bg-dark-2` вместо классов Tailwind, которые используют CSS переменные.

**Пример проблемы**:
```tsx
// ❌ Неправильно
<div className="bg-white dark:bg-dark-2">

// ✅ Правильно  
<div className="bg-background dark:bg-surface">
// или еще лучше через CSS переменные напрямую
<div className="bg-[rgb(var(--color-background))] dark:bg-[rgb(var(--color-surface))]">
```

**Файлы для исправления**:
- `frontend/src/components/Dashboard/ProfileCard.tsx`
- `frontend/src/components/Dashboard/WidgetSelector.tsx`
- `frontend/src/components/Modals/*.tsx`
- И другие компоненты в `frontend/src/components/`

### 2. Неполное использование i18n

**Проблема**: Некоторые компоненты имеют hardcoded тексты вместо использования `t()`.

**Пример проблемы**:
```tsx
// ❌ Неправильно
<span>Управление аккаунтом</span>

// ✅ Правильно
<span>{t('profile.accountManagement', 'Управление аккаунтом')}</span>
```

**Файлы для проверки**:
- `frontend/src/design-system/composites/ProfilePopup/ProfilePopup.tsx` - есть hardcoded тексты

### 3. Переиспользование компонентов

**Проблема**: Некоторые компоненты создают свои стили вместо использования компонентов из дизайн-системы.

**Пример**: В модалках используется нативный `<select>` вместо компонента Select из дизайн-системы (если такой есть).

## План исправлений

### Этап 1: Унификация использования темы
1. Заменить все `dark:bg-dark-2` на `bg-surface`
2. Заменить все `dark:text-white` на `text-text-primary`
3. Заменить все `dark:text-dark-6` на `text-text-secondary`
4. Заменить все `dark:border-dark-3` на `border-border`

### Этап 2: Добавление недостающих переводов
1. Проверить все компоненты на наличие hardcoded текстов
2. Добавить переводы в файлы локализации
3. Заменить hardcoded тексты на `t()`

### Этап 3: Унификация компонентов
1. Проверить использование нативных элементов (`<select>`, `<input type="file">`)
2. Заменить на компоненты из дизайн-системы где возможно
3. Создать недостающие компоненты в дизайн-системе если нужно
